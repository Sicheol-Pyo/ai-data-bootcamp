## 4. 처리 결정과 근거 (5줄로)
1. 중복 행 765건 유지함→ 사유: 해당 데이터의 경우 판매 날짜(date_str)만 포함되어 있고 별도 판매 시간 관련 column이 존재하지 않음.
이에 특정 매장(store_id)에 판매된 날짜(date_str) 기준 동일한 품목(item)이 여러 번 판매될 수 있어 중복 처리는 별도로 진행하지 않았으며, 추후 판매시간 관련 자료 추가 수집이 필요할 것으로 사료됨.

2. store_id 공백 → ' S01' -> 'S01'로 str.strip() 처리함.
3. item '케익' → '케이크' → '케익'으로 표기 혼재된 10건 '케이크'로 통일.
4. revenue 음수 → 결측으로 전환 후 아래 결측 처리에 따라 처리함.
5. revenue 결측 61건(음수 처리 1건) → 제거 (이유: quantity, unit_price 값이 보존되어 quantity * unit_price 계산값으로 대체함.)
6. date_str parsing → date 서식 통일을 위하여 format = "mixed"로 parsing 진행함.
